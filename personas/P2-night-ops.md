# P2: 深夜障害対応をする運用者としての自分 (Secondary)

> マスター: `../personas.md` §4.1

## 一言サマリ

深夜に Discord で起こされた P1。「朝まで安全に待てる状態」にして寝直すことが目的。深い判断を拒否する。

## 場面 / Device / Surface

- 場所: ベッドサイド / 外出先の深夜帯。
- Device: スマホ (Discord)。
- PC / runner-cli / Forgejo は原則使わない。

## 認知予算

- 介入 1 回あたり 1 分以内。
- 2–3 アクションで終わらせたい。
- 複雑な手順を記憶・実行する能力は低い。

## 最重要ニーズ

- 障害の種類が局所か全体かを即判別できる。
- in-flight task と停止済み task を区別できる。
- 強制介入コマンドが少数で、意味が明確。
- 復旧後の再開ルールが機械的に決まる。

## 成功

- 通知 1 通で「今動くか、朝まで寝てよいか」が判別できる。
- 動くべき場合でも 2–3 アクションで完了する。
- 無介入でも task-relay が自律的に安全停止する。
- 翌朝の P1 が状況把握できる情報が残る。

## 失敗

- 連続通知で何度も起こされる。
- 通知に重要度の判別材料がない。
- 半覚醒で長文コマンドを打たされる。
- 状況が勝手に変わり翌朝に再構成不能。

## 代表 utterance

- 「これ朝まで待てる?」
- 「止まってる? 放っといていい?」
- 「明日やる。今は寝たい」

## 設計含意

- breaker / degrade / cancel / unlock は意味を混ぜない。
- 停止系通知には「何が止まったか」「人間がやるべきこと」を含める。
- 強制コマンドは admin 限定、少数、可逆性の低さを明記する。
